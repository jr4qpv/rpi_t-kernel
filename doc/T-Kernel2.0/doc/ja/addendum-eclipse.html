1. Eclipse 開発環境を他機種用に使用する方法

本パッケージに含まれている Eclipse 開発環境のプラグインは、機種 "tef_em1d" 用です。他の機種で Eclipse を利用にするには、その機種に対応したプラグインを作成する必要がありますが、以下の方法により、簡易的に他の機種に対応した Eclipse を用意することも可能です。

ここでは、機種名 "bbb_cccc" に対応した Eclipse を用意するものとします。条件として、ソースディレクトリの構成は "tef_em1d" と同一であり (機種名の差異によるディレクトリ名の差異を除く)、Makefile 等は機種名 "bbb_cccc" に合わせて修正済みとします。

  1. (1) ビルド環境名 "tef_em1d" をそのまま使用する方法

    最も簡単な方法で、以下の操作だけで対応できます。ただし、ビルド環境名として、実際の機種名とは異なる "tef_em1d" を使用する必要があります。

    • Eclipse を起動し、ワークスペースとして、機種 "bbb_cccc" 用の以下のフォルダを指定します。

      • C:\cygwin\usr\local\bbb_cccc\tkernel_source

    • メニュー「ウィンドウ」→「設定」で、機種 "bbb_cccc" 用に以下の設定を行います。

      • 「環境変数」 "BD", "GNU_BD", "GNUARM_2" のパス名中の "tef_em1d" を "bbb_cccc" に変更する。

  2. (2) 機種に対応したビルド環境名 "bbb_cccc" を追加する方法

    簡易的にプラグインの一部を修正することにより、ビルド環境名として機種名 "bbb_cccc" を使用できるようになります。

    • "C\:eclipse\plugins" フォルダ内のフォルダ "org.t_engine.tl.tef_em1d.1.1.2_1.1.2" 全体をコピーして "org.t_engine.tl.bbb_cccc.1.1.2_1.1.2" の名称に変更します。

    • コピーした "org.t_engine.tl.bbb_cccc.1.1.2_1.1.2" フォルダ内の以下のファイルを修正します。

      • ファイル - plugin.xml

        name="tef_em1d.1.1.2" を以下に変更 name="bbb_cccc.1.1.2"

      • ファイル - META-INF\MANIFEST.MF

        Bundle-Name: tef_em1d Plug-in
  を以下に変更
Bundle-Name: bbb_cccc Plug-in

Bundle-SymbolicName: org.t_engine.tl.tef_em1d.1.1.2;singleton:=true
  を以下に変更
Bundle-SymbolicName: org.t_engine.tl.bbb_cccc.1.1.2;singleton:=true

他の部分の "tef_em1d" は変更してはいけません。

    • "C:\eclipse" フォルダ内の "eclipse.exe -clean.cmd" をダブルクリックして、Eclipse を初期化起動し、ワークスペースとして、機種 "bbb_cccc" 用の以下のフォルダを指定します。

      • C:\cygwin\usr\local\bbb_cccc\tkernel_source

    • メニュー「ウィンドウ」→「設定」で、「T-Engine 開発環境」の下に "tef_em1d.1.1.2" と "bbb_cccc.1.1.2" の両方が表示されていることを確認した後、"bbb_cccc.1.1.2" を選択して、機種 "bbb_cccc" 用に以下の設定を行います。

      • 「環境変数」 "BD", "GNU_BD", "GNUARM_2" のパス名中の "tef_em1d" を "bbb_cccc" に変更する。

      • 「出力ディレクトリ」は、"bbb_cccc" に変更してもしなくてもどちらでも構いません。

    • 新規プロジェクト生成時の「ターゲット」を "bbb_cccc.1.1.2" とします。

    • ビルド先ディレクトリの「プロパティ」の「ビルド環境」を "bbb_cccc.1.1.2" とします。
