1. Cygwin Installation

The GNU development environment used as the T-Kernel development environment runs on Windows Cygwin, so Cygwin must be installed first. Install Cygwin using the following procedure.

  • The environment does not run properly on Cygwin 1.5 or older versions. Reinstall the latest Cygwin version if you are using an older version.

  • To install Cygwin, log on to Windows with a user account that has administrator privileges.

  1. (1) Copy the Cygwin install file "cygwin_d-1.7.17-1.zip" located in the "develop" folder of this package to the drive where it is to be installed (Choose a temporary directory on the drive, e.g. "C:\temp"). To continue, right-click the copied file in the Windows "Explorer" and run "Extract All".

  2. (2) The "setup.exe" installer is located in the extracted folder. Double-click "setup.exe" to start the installer.

  3. (3) When the installer window opens, press "Next (N)>".

  4. (4) Select "Install from Local Directory" and press "Next (N)>".

  5. (5) Specify the location to set as the Cygwin root directory on Windows. The default root directory is "C:\cygwin". If there are no issues, please use the default.

  6. (6) Select the default "All Users" for the "Install For" option. Press "Next (N)>".

  7. (7) Specify the folder where "cygwin_d-1.7.17-1.zip" was extracted as the "Local Package Directory". Press "Next (N)>" to check the files.

  8. (8) After a few moments, the "Setup Alert" dialog is displayed when first installing Cygwin 1.7. There may be adverse effects when overwriting the installation of a previous version of Cygwin, so please read the WARNING before proceeding.

  9. (9) Next, the installation package selection screen is displayed. Click "Default" for "All () Default", set to "Install", and press "Next (N)>" to start the installation.

    Press "Next (N)>" when the "Resolving Dependencies : W32api" dialog is displayed.

  10. (10) Finally, the installer will ask if you wish to create Desktop and Start Menu icons. Press "Finish" to exit the installer.

  11. (11) Depending on the Windows version, a screen reading "This program might not have installed correctly" may be displayed. In this case, press "This program installed correctly".

  12. (12) The installation process above, installs the minimum requirements to run the GNU development environment. If other functions are required, run "setup.exe" again, and select "Install from Internet" to add the necessary functions.

2. Cygwin Post-installation Settings

After the Cygwin installation finishes, modify the batch file for starting Cygwin using the administrator privileges used to perform the installation.

The Cygwin root directory is "C:\cygwin" on Windows. For example, the directory shown as "/usr" in Cygwin corresponds to the folder "C:\cygwin\usr" on Windows Explorer.

  1. (1) Setting to distinguish between upper and lowercase characters

    The default Cygwin setting does not distinguish between upper and lowercase characters, which may cause a problem, so we will change the setting to make it case-sensitive.

    Use a text editor to open the "C:\cygwin\Cygwin.bat" batch file which is run during Cygwin start up.

        bash --login -i

    Add the following line before the line above.

        set CYGWIN=check_case:strict

    After modifying the batch file, double-click the Cygwin icon on the Desktop or the "Cygwin.bat" file in the "C:\cygwin" folder. Cygwin starts up and opens the Cygwin console.

    All Cygwin operations are performed by entering commands in the bash command shell in the Cygwin console.

    Enter "exit" in the Cygwin console to exit Cygwin.

        $ exit

If the installation user is not the Cygwin user, please log off from Windows, log on as the Cygwin user, and launch Cygwin. The user's "home" directory is initialized when Cygwin is started for the first time.

If required, edit the "/home/username/.bashrc" bash startup command file to make it easier.

Please delete the "cygwin_d-1.7.17-1.zip" file copied to "C:\temp" after installation as it is no longer needed.
