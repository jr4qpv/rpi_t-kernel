1. How to use the Eclipse development environment for other models

The Eclipse development environment plugin included with this package is intended for use with the "tef_em1d" model. To use Eclipse with other models, a plugin which supports that model must be created, but Eclipse can also be easily set up to support other models using the following method.

In this example, we will set up Eclipse to use a model named "bbb_cccc". As a requirement, the source directory configuration is the same as "tef_em1d" (except where the directory name is different due to the difference in model name), and the name of the Makefile, etc. have been edited to match the model name "bbb_cccc".

  1. (1) How to use the build environment name "tef_em1d" as is

    The simplest method is to use the following operation. However, you must use the build environment name "tef_em1d" which differs from the actual model name.

    • Start Eclipse and specify the following folder as the work space for the "bbb_cccc" model.

      • C:\cygwin\usr\local\bbb_cccc\tkernel_source

    • Enter the following settings from the "Window" -> "Preferences" menu for the "bbb_cccc" model.

      • Change "tef_em1d" to "bbb_cccc" in the path names for the "Environment Variables" "BD", "GNU_BD", and "GNUARM_2".

  2. (2) How to add the "bbb_cccc" build environment name that corresponds with the model

    The "bbb_cccc" model name can be used as the build environment name by easily modifying part of the plugin.

    • Go to the folder "C\:eclipse\plugins", copy everything in "org.t_engine.tl.tef_em1d.1.1.2_1.1.2" and change the name to "org.t_engine.tl.bbb_cccc.1.1.2_1.1.2".

    • Modify the following files in the "org.t_engine.tl.bbb_cccc.1.1.2_1.1.2" folder which was copied.

      • File - plugin.xml

        name="tef_em1d.1.1.2" change to the following name="bbb_cccc.1.1.2"

      • File - META-INF\MANIFEST.MF

        Bundle-Name: tef_em1d Plug-in
  change to the following
Bundle-Name: bbb_cccc Plug-in

Bundle-SymbolicName: org.t_engine.tl.tef_em1d.1.1.2;singleton:=true
  change to the following
Bundle-SymbolicName: org.t_engine.tl.bbb_cccc.1.1.2;singleton:=true

Do not change any other instances of "tef_em1d".

    • Double-click "eclipse.exe -clean.cmd" in the "C:\eclipse" folder, initialize and start Eclipse, and specify the following folder as the work space for the "bbb_cccc" model.

      • C:\cygwin\usr\local\bbb_cccc\tkernel_source

    • Go to the "Window" -> "Preferences" menu and confirm that both "tef_em1d.1.1.2" and "bbb_cccc.1.1.2" are displayed under "T-Engine Development Environment". Select "bbb_cccc.1.1.2" and enter the following settings for the "bbb_cccc" model.

      • Change "tef_em1d" to "bbb_cccc" in the path names for the "Environment Variables" "BD", "GNU_BD", and "GNUARM_2".

      • The "Output Directory" does not need to be changed to "bbb_cccc".

    • Set the "Target" when generating a new project to "bbb_cccc.1.1.2".

    • Set the "Build Environment" under the build directory "Properties" to "bbb_cccc.1.1.2".
