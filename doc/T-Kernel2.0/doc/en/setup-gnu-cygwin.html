GNU Development Environment Settings (Cygwin use)

1. Creating the Work Directory

First, create the directory to install the GNU development environment. This example installs into "/usr/local/tef_em1d".

Run the following command on the Cygwin console to create the directory.

    $ mkdir /usr/local/tef_em1d

  • If another T-Kernel GNU development environment is already installed, choose a different installation directory. Installing into the same directory may cause problems to occur.

  • Typically, the "/usr/local" owner is the administrator who installed Cygwin, so administrator privileges are needed to create "/usr/local/tef_em1d". After creating "/usr/local/tef_em1d", set the owner and permissions properly so that the user who performs the T-Kernel development work can access the directory.

2. Extracting the Development Environment Package

Next, extract the GNU development environment into "/usr/local/tef_em1d".

First, use the Windows Explorer, etc. to copy the following files in the "develop" folder of this package to the "C:\cygwin\usr\local\tef_em1d" folder.

    - te.Cygwin-i686.arm_2.1.0.3.tar.gz
    - te.Cygwin-i686.common.15.tar.gz

Next, run the following commands in sequence on the Cygwin console to extract each package. This sets the GNU development environment to "/usr/local/tef_em1d/tool".

    $ cd /usr/local/tef_em1d
    $ tar zxpf te.Cygwin-i686.arm_2.1.0.3.tar.gz
    $ tar zxpf te.Cygwin-i686.common.15.tar.gz

The Windows drive "C:\" can be accessed as "/cygdrive/C/" from the Cygwin console which enables the files to be extracted by directly specifying the Windows file path without copying the tar archives to the destination drive's directory.

    Example: $ tar zxpf /cygdrive/C/tmp/te.Cygwin-i686.arm_2.1.0.3.tar.gz

Delete the copied tar.gz file after extraction as they are no longer needed.

3. Setting Environment Variables

To continue, set the required environment variables.

Edit the bash launch command file "/home/username/.bashrc" ( "C:\cygwin\home\username\.bashrc") and add the following setting commands.

  • Environment variable to compile and build - GNU_BD, GNUARM_2

    export GNU_BD=/usr/local/tef_em1d/tool/Cygwin-i686
export GNUARM_2=$GNU_BD/arm_2-unknown-tkernel

  • Environment variable that represents the T-Kernel source's top directory - BD

    export BD=/usr/local/tef_em1d/tkernel_source

Change the path name if required to match the path name of the work directory where the GNU development environment was installed.

The .bashrc setting changes are valid after cygwin is rebooted. Run the following command on the Cygwin console to immediately enable the changes.

    $ source ~/.bashrc

4. Extracting the T-Kernel Source Code Package

Use the Windows Explorer, etc. to copy the following files in the "srcpkg" folder of this package to the "C:\cygwin\usr\local\tef_em1d" folder.

    - tkernel_source.tar.gz

Next, run the following commands on the Cygwin console to extract the package. The T-Kernel source code package is extracted to "/usr/local/tef_em1d/tkernel_source".

    $ cd /usr/local/tef_em1d
    $ tar zxpf tkernel_source.tar.gz

Delete the copied tkernel_source.tar.gz file after extraction as it is no longer needed.
