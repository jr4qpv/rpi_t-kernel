1. Creating the Work Directory

First, create the directory to install the GNU development environment. This example installs into "/usr/local/tef_em1d".

Run the following command from the Linux console to create the directory.

    $ mkdir /usr/local/tef_em1d

  • If another T-Kernel GNU development environment is already installed, choose a different installation directory. Installing into the same directory may cause problems to occur.

  • Typically, the "/usr/local" owner is "root", so root privileges are needed to create "/usr/local/tef_em1d". Use the "sudo" command if you are not logged in with root privileges. After creating "/usr/local/tef_em1d", set the appropriate owner and permissions so that the user who performs the T-Kernel development work can access the directory.

2. Extracting the Development Environment Package

Next, extract the GNU development environment into "/usr/local/tef_em1d".

First, copy the following files in the "develop" directory of this package to the "/usr/local/tef_em1d" directory.

    - te.Linux-i686.arm_2.1.0.3.tar.gz
    - te.Linux-i686.common.15.tar.gz

Next, run the following commands in sequence from the Linux console to extract the package. This sets the GNU development environment to "usr/local/tef_em1d/tool".

    $ cd /usr/local/tef_em1d
    $ tar zxpf te.Linux-i686.arm_2.1.0.3.tar.gz
    $ tar zxpf te.Linux-i686.common.15.tar.gz

Delete the copied tar.gz file after extraction as they are no longer needed.

3. Preparing the Development Tools

The following tools are required to use the GNU development environment. If they are not installed on your Linux system, please download and make sure they are in a usable state.

    - GNU Make
    - perl

4. Environment Variable Settings

To continue, set the required environment variables.

Edit the bash launch command file "/home/username/.bashrc" and add the following setting commands.

  • Environment variable to compile and build - GNU_BD, GNUARM_2

    export GNU_BD=/usr/local/tef_em1d/tool/Linux-i686
export GNUARM_2=$GNU_BD/arm_2-unknown-tkernel

  • Environment variable that represents the T-Kernel source's top directory - BD

    export BD=/usr/local/tef_em1d/tkernel_source

Change the path name if required to match the path name of the work directory where the GNU development environment was installed.

The .bashrc setting changes are valid after the console is rebooted. Run the following command on the console to immediately enable the changes.

    $ source ~/.bashrc

5. Extracting the T-Kernel Source Code Package

First, copy the following files in the "srcpkg" directory of this package to the "/usr/local/tef_em1d" directory.

    - tkernel_source.tar.gz

Next, run the following commands from the Linux console to extract the package. The T-Kernel source code package is extracted to "/usr/local/tef_em1d/tkernel_source".

    $ cd /usr/local/tef_em1d
    $ tar zxpf tkernel_source.tar.gz

Delete the copied tkernel_source.tar.gz file after extraction as it is no longer needed.
