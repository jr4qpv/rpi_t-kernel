1. Eclipse System Installation

This section installs the Eclipse + CDT (C/C++ Development Tooling) environment.

Please log in to Windows with the account which will actually use Eclipse and perform the following tasks.

Copy the following files located in the "develop" folder of this package to an appropriate folder on the drive where it is to be installed (e.g. "C:\temp").

    - Eclipse + CDT             "eclipse-cpp-indigo-SR2-incubation-win32.zip"
    - Eclipse Japanese plugin   "pleiades_1.3.4.zip"
  • There are newer versions of Eclipse on the Eclipse home page, but compatibility testing was performed with the version above. Please use this version of Eclipse (Indigo SR2).

Next, extract each of the zip files above in the order in which they are listed. Select and right-click the files above in the Explorer and select "Extract All". Specify the extraction location as follows.

    - Eclipse + CDT              "C:\"
    - Eclipse Japanese plugin    "C:\eclipse"

Eclipse is extracted into the "C:\eclipse" folder.

  • Do not install with a path name that includes spaces or Japanese characters. The software will not operate properly if there is a space in the path name (e.g. "C:\Program Files").
  • The Eclipse Japanese plugin does not need to be extracted if you are using a non-Japanese environment.
  • The system will ask if you wish to combine the folders when extracting the "Eclipse Japanese plugin". Press "Yes".

To use Eclipse in a Japanese environment, add the following statement to the last line of "C:\eclipse\eclipse.ini". Delete this line to change the environment back to English.

    -javaagent:plugins/jp.sourceforge.mergedoc.pleiades/pleiades.jar

Use Explorer to open the "C:\eclipse" folder where Eclipse is installed. The "eclipse.exe" file is the Eclipse program. Create a shortcut to this file on the Desktop.

Please delete the zip files copied to "C:\temp" after installation as they are no longer needed.

2. T-Kernel Development Plugin Installation

To continue, install the T-Kernel development plugin.

Copy the following files located in the "develop" folder of this package to an appropriate folder on the drive where it is to be installed (e.g. "C:\temp").

  - Common platform elements     "org.t_engine.te.1.1.2.zip"
  - tef_em1d support elements    "org.t_engine.tl.tef_em1d.1.1.2.zip"
  • If you are using plugins from a previous version, move them to another folder or delete them.

Next, extract each of the zip files above in the order in which they are listed. Select and right-click the files above in the Explorer and select "Extract All". Specify the extraction location as "C:\eclipse".

Please delete the zip files copied to "C:\temp" after installation as they are no longer needed.

3. Other Settings

If the Windows HOME environment variable is set, it will conflict with the Cygwin environment variable, and Eclipse will not run properly. Change the Windows HOME environment variable to another name (Ex. HOME_ORG) or delete it.

  • Access and edit the Windows environment variables under "Control Panel" -> ("System and Security" ->) "System" -> "Advanced system settings" -> "Environment variables".
